## Запрос перевода проекта

Если вы хотите внести перевод или вам нужен перевод проекта:

1. Создайте issue по следующему шаблону:

```md
**Проект**: [project_url]
**Целевой язык**: [target_lang]
**Описание**: Краткое описание ценности этого перевода
```

2. Рабочий процесс:

```mermaid
sequenceDiagram
  Участник->>Проект Переводчик: Создать issue о переводе
  Проект Переводчик->>Сообщество: Проверить issue
  Сообщество-->>Участник: Одобрить/Прокомментировать
  Участник->>Новый проект: Начать перевод
  Участник->>Новый проект: Отправить в новый проект
  Участник->>Проект Переводчик: Создать Pull Request, изменить README.Samples
  Проект Переводчик-->>Проект Переводчик: Проверить и объединить
```

3. После объединения PR перевод будет добавлен в раздел Примеры.

Текущие переводы в процессе: [Просмотреть Issues](https://github.com/Project-Translation/project_translator/issues)

## Функции

- 📁 Поддержка перевода на уровне папок
  - Перевод целых папок проекта на несколько языков
  - Сохранение оригинальной структуры папок и иерархии
  - Поддержка рекурсивного перевода подпапок
  - Автоматическое обнаружение переводимого контента
  - Пакетная обработка для эффективного перевода больших объемов
- 📄 Поддержка перевода на уровне файлов
  - Перевод отдельных файлов на несколько языков
  - Сохранение оригинальной структуры и форматирования файлов
  - Поддержка режимов перевода папок и файлов
- 💡 Умный перевод с ИИ
  - Автоматическое сохранение целостности кодовой структуры
  - Перевод только комментариев кода, сохранение логики кода
  - Сохранение форматов JSON/XML и других структур данных
  - Профессиональное качество перевода технической документации
- ⚙️ Гибкая конфигурация
  - Настройка исходной папки и нескольких целевых папок
  - Поддержка пользовательских интервалов перевода файлов
  - Настройка игнорируемых типов файлов
  - Поддержка нескольких вариантов ИИ моделей
- 🚀 Удобство использования
  - Отображение прогресса перевода в реальном времени
  - Поддержка паузы/возобновления/остановки перевода
  - Автоматическое поддержание структуры целевой папки
  - Инкрементный перевод для избежания дублирования работы
- 🔄 Расширенный перевод с применением сравнения
  - **Точное обновление**: Переводится только реально измененный контент
  - **Контроль по строкам**: Точное сохранение форматирования и структуры
  - **Инкрементный перевод**: Идеально для поддержания переведенной документации
  - **Удобство для контроля версий**: Генерирует минимальные, целевые изменения
  - **Экономичность**: Сокращает вызовы API за счет перевода только различий
  - **Автоматическое резервное копирование**: Автоматически создает резервные копии перед применением изменений

## Конфигурация

Расширение поддерживает следующие параметры конфигурации:

```json
{
  "projectTranslator.specifiedFolders": [
    {
      "sourceFolder": {
        "path": "Путь к исходной папке",
        "lang": "Код исходного языка"
      },
      "targetFolders": [
        {
          "path": "Путь к целевой папке",
          "lang": "Код целевого языка"
        }
      ]
    }
  ],
  "projectTranslator.diffApply": {
    "enabled": true,
    "validationLevel": "normal",
    "autoBackup": true,
    "maxOperationsPerFile": 100
  },
  "projectTranslator.specifiedFiles": [
    {
      "sourceFile": {
        "path": "Путь к исходному файлу",
        "lang": "Код исходного языка"
      },
      "targetFiles": [
        {
          "path": "Путь к целевому файлу",
          "lang": "Код целевого языка"
        }
      ]
    }
  ],
  "projectTranslator.currentVendor": "openai",
  "projectTranslator.vendors": [
    {
      "name": "openai",
      "apiEndpoint": "URL конечной точки API",
      "apiKey": "Ключ аутентификации API",
      "model": "Имя используемой модели",
      "rpm": "Максимальное количество запросов в минуту",
      "maxTokensPerSegment": 4096,
      "timeout": 30,
      "temperature": 0.0
    }
  ]
}
```

Основные параметры конфигурации:

| Параметр конфигурации                        | Описание                                                                                    |
| ------------------------------------------- | ---------------------------------------------------------------------------------------------- |
| `projectTranslator.specifiedFolders`        | Несколько исходных папок со своими целевыми папками для перевода           |
| `projectTranslator.specifiedFiles`          | Несколько исходных файлов со своими целевыми файлами для перевода               |
| `projectTranslator.translationIntervalDays` | Интервал перевода в днях (по умолчанию 7 дней)                                                  |
| `projectTranslator.copyOnly`                | Файлы для копирования без перевода (с массивами `paths` и `extensions`)                         |
| `projectTranslator.ignore`                  | Игнорируемые файлы (с массивами `paths` и `extensions`)                              |
| `projectTranslator.currentVendor`           | Текущий используемый поставщик API                                                                      |
| `projectTranslator.vendors`                 | Список конфигураций поставщиков API                                                                  |
| `projectTranslator.systemPrompts`           | Массив системных подсказок для направления процесса перевода                                        |
| `projectTranslator.userPrompts`             | Пользовательские подсказки, которые добавляются после системных подсказок во время перевода |
| `projectTranslator.segmentationMarkers`     | Маркеры сегментации, настроенные по типу файла, поддерживают регулярные выражения                     |
| `projectTranslator.debug`                   | Включить режим отладки для логирования всех запросов и ответов API в выходной канал (по умолчанию: false)    |
| `projectTranslator.logFile`                 | Конфигурация файлов журналов (см. [Функция журналирования](./docs/log-file-feature.md))        |

## Использование

1. Открыть палитру команд (Ctrl+Shift+P / Cmd+Shift+P)
2. Ввести "Translate Project" и выбрать команду
3. Если исходная папка не настроена, появится диалог выбора папки
4. Дождаться завершения перевода

Во время перевода:

- Можно приостанавливать/возобновлять через кнопки в статусной строке
- Можно остановить процесс в любое время
- Прогресс отображается в уведомлениях
- Подробные логи в панели вывода

## Разработка

### Система сборки

Это расширение использует esbuild для быстрой сборки и разработки:

#### Доступные скрипты

- `npm run build` - Сборка для продакшена с минификацией
- `npm run compile` - Сборка для разработки
- `npm run watch` - Режим наблюдения для разработки
- `npm test` - Запуск тестов

#### Задачи в VS Code

- **Build** (Ctrl+Shift+P → "Tasks: Run Task" → "build") - Сборка расширения для продакшена
- **Watch** (Ctrl+Shift+P → "Tasks: Run Task" → "watch") - Режим разработки с автоматической пересборкой

### Настройка разработки

1. Клонировать репозиторий
2. Выполнить `npm install` для установки зависимостей
3. Нажать `F5` для запуска отладки или выполнить задачу "watch" для разработки

Конфигурация esbuild:

- Объединяет все TypeScript файлы в один `out/extension.js`
- Исключает VS Code API (помечены как внешние)

## Расширенные функции

### Перевод с применением сравнения

Для подробной информации о расширенном режиме перевода см. [Руководство по использованию Diff Apply](./docs/diff-apply-usage.md).

Эта функция обеспечивает:
- Точный построчный перевод изменений
- Снижение затрат API для больших файлов
- Лучшую интеграцию с контролем версий
- Сохранение форматирования документов

### Документация по проектированию

Для технических деталей реализации diff apply см. [Проектирование перевода с применением сравнения](./docs/diff-apply-translation-design.md).
- Генерирует source map для сборок разработки
- Минифицирует код для продакшена
- Предоставляет интеграцию с проблемными матчами для VS Code
